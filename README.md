# 🚀 Quantum Leap

> Персональный дневник трансформации с ИИ-анализом

Quantum Leap — это приложение для отслеживания личного роста. Ты фиксируешь «Я сейчас» и «Я хочу стать», каждый день заполняешь чекин, а в конце месяца получаешь ИИ-сводку о том, что изменилось.

## ✨ Особенности

- 📝 **Ежедневные чекины** — 5 вопросов о настроении, мыслях и действиях
- 👤 **Профиль трансформации** — зафиксируй точку А и точку Б своего пути
- 🤖 **ИИ-анализ** — месячная сводка с честной обратной связью
- 💾 **Автоматическое сохранение** — все данные хранятся в браузере
- 🔒 **Полная приватность** — никаких облачных сервисов
- 🎨 **Минималистичный дизайн** — ничего лишнего, только суть

## 🛠️ Установка

### Шаг 1: Скачай проект

```bash
git clone https://github.com/твой-username/quantum-leap.git
cd quantum-leap
```

### Шаг 2: Установи локальный ИИ (Ollama)

1. Скачай Ollama: https://ollama.com/download
2. Установи модель:
   ```bash
   ollama pull llama3.2
   ```
3. Проверь установку:
   ```bash
   ollama list
   ```

**Рекомендуемые модели:**
- `llama3.2` (3B) — быстрая, работает на слабых ПК
- `mistral` (7B) — баланс скорости и качества
- `gemma2` (9B) — отличное качество анализа

### Шаг 3: Настрой конфигурацию

Файл `js/config.js` уже создан из примера. Если нужно изменить настройки:

```javascript
const CONFIG = {
    LOCAL_AI_URL: 'http://localhost:11434/v1',  // URL Ollama
    LOCAL_AI_MODEL: 'llama3.2',                 // Название модели
    AI_MAX_TOKENS: 2000,                        // Макс. длина ответа
    AI_TEMPERATURE: 0.7                         // Креативность (0-1)
};
```

**Для LM Studio:**
- Измени `LOCAL_AI_URL` на `http://localhost:1234/v1`
- Укажи название загруженной модели

### Шаг 4: Запусти сервер (опционально)

Если хочешь, чтобы данные автоматически сохранялись в файл `notes.txt`:

```bash
# Установи зависимости
npm install

# Запусти сервер
npm start
```

Сервер запустится на `http://localhost:3000` и будет работать в фоне. Все данные будут автоматически дописываться в файл `notes.txt`.

**Если не запускаешь сервер** — всё равно работает! Данные будут сохраняться только в браузере (localStorage).

### Шаг 5: Открой приложение

Просто открой `index.html` в браузере:
- **Windows:** Двойной клик по файлу
- **macOS:** `open index.html`
- **Linux:** `xdg-open index.html`

## 📖 Как пользоваться

### Первый запуск

1. **Открой приложение**
   - Просто открой `index.html` в браузере
   - Данные автоматически начнут сохраняться в localStorage браузера

2. **Заполни профиль**
   - Перейди в таб «Мой профиль»
   - Заполни блок «Я сейчас» (5 вопросов)
   - Заполни блок «Кем хочу стать» (5 вопросов)
   - Данные сохранятся автоматически

3. **Начни делать чекины**
   - Каждый день заполняй чекин (5 вопросов)
   - Данные сохраняются автоматически при каждом изменении
   - Календарь покажет, в какие дни ты заполнял чекины

### Где хранятся данные?

Все данные хранятся **локально в браузере** (localStorage). Это значит:
- ✅ Никто кроме тебя не имеет доступа к данным
- ✅ Данные не отправляются в интернет
- ✅ Всё работает даже без интернета (кроме ИИ-анализа)
- ⚠️ Если очистишь данные браузера — потеряешь записи

**Резервное копирование:**
- Нажми кнопку **«💾 Скачать резервную копию»** в шапке сайта
- Файл `quantum-leap-answers.txt` скачается в папку Загрузки
- Храни его в безопасном месте (облако, флешка)

### Генерация сводки

**Вариант 1: Локальный ИИ (Ollama)**

В конце месяца (или в любой момент для теста):
1. Перейди в таб «Сводка»
2. Нажми «Сгенерировать сейчас (тест)»
3. Подожди, пока ИИ проанализирует твой месяц
4. Прочитай сводку

**Вариант 2: Внешний ИИ (ChatGPT, Claude, Gemini)**

Если не хочешь использовать локальный ИИ:
1. Нажми кнопку **«📤 Экспорт для ИИ»** в шапке сайта
2. Файл `my-data.txt` скачается автоматически
3. Открой файл `export-for-ai/master-prompt.txt`
4. Скопируй мастер-промт
5. Открой ChatGPT, Claude или любую другую нейросеть
6. Вставь мастер-промт
7. Затем вставь содержимое файла `my-data.txt`
8. Получи анализ своего месяца!

Этот способ удобен, если:
- У тебя нет мощного компьютера для локального ИИ
- Ты хочешь использовать конкретную модель (GPT-4, Claude Opus и т.д.)
- Тебе нужен более детальный анализ

## 🖥️ Режим с сервером (опционально)

Приложение работает в двух режимах:

### Режим 1: Только браузер (по умолчанию)
- Данные хранятся в localStorage браузера
- Никаких серверов не нужно
- Просто открой `index.html` и работай

### Режим 2: С локальным сервером
- Данные сохраняются **и** в браузере, **и** в файл `notes.txt`
- Удобно для резервного копирования
- Можно читать данные из файла напрямую

**Как включить серверный режим:**

```bash
# 1. Установи зависимости
npm install

# 2. Запусти сервер
npm start
```

Сервер запустится на `http://localhost:3000` и будет работать в фоне. Теперь при каждом сохранении данные будут автоматически дописываться в файл `notes.txt`.

**Что делает сервер:**
- Принимает данные от фронтенда
- Дописывает их в файл `notes.txt` (не перезаписывает!)
- Работает тихо в фоне, не мешает

**Если сервер не запущен:**
- Приложение работает как обычно
- Данные сохраняются только в браузере
- Никаких ошибок не будет

## 🔧 Технические детали

### Стек

- **Frontend:** HTML + CSS + Vanilla JS
- **Backend (опционально):** Node.js + Express
- **Хранение:** localStorage (браузер) + notes.txt (сервер)
- **ИИ:** Ollama / LM Studio (локальная модель)
- **Формат данных:** Простой текстовый файл

...

## 🐛 Проблемы?

### ИИ не отвечает

1. Проверь, запущен ли Ollama: `ollama list`
2. Проверь URL в `config.js`
3. Убедись, что модель установлена: `ollama pull llama3.2`

### Данные пропали

1. Проверь localStorage браузера (F12 → Application → Local Storage)
2. Найди последнюю резервную копию `quantum-leap-answers.txt` в папке Загрузки
3. Если очистил данные браузера — восстанови из резервной копии

### Браузер тормозит

localStorage имеет ограничение ~5-10 МБ. Если записей очень много:
1. Скачай резервную копию
2. Очисти localStorage (F12 → Application → Local Storage → Clear)
3. Начни заново или импортируй только нужные записи

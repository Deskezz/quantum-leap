# 📋 Техническое задание: Quantum Leap

> Персональный дневник трансформации с ИИ-анализом. Пользователь фиксирует «Я сейчас» и «Я хочу стать», каждый день заполняет чекин, а в начале каждого месяца получает ИИ-сводку о том, что изменилось.

---

## 1. Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| Frontend | HTML + CSS + Vanilla JS | Максимальная скорость, нет лишних зависимостей |
| База данных | Supabase (PostgreSQL) | Бесплатный тир: 500 MB, без ограничений по времени |
| ИИ-анализ | Настраивается через `config.js` | OpenAI API **или** локальная модель через OpenAI-совместимый API |
| Хостинг | GitHub Pages / Netlify | Статический сайт, бесплатно |
| Версионирование | Git + GitHub | `.gitignore` скрывает все ключи |

## 3. Файл конфига (config.example.js)

Этот файл **коммитится в GitHub**. Пользователь копирует его в `config.js` и вставляет свои значения.

```javascript
// Скопируй этот файл в config.js и заполни своими значениями
const CONFIG = {
  // --- Supabase ---
  SUPABASE_URL: 'ВАШ_SUPABASE_URL',           // https://xxxx.supabase.co
  SUPABASE_ANON_KEY: 'ВАШ_SUPABASE_ANON_KEY', // eyJ...

  // --- ИИ: выбери режим ---
  AI_MODE: 'openai', // 'openai' или 'local'

  // Режим OpenAI
  OPENAI_API_KEY: 'ВАШ_OPENAI_API_KEY',       // sk-...
  OPENAI_MODEL: 'gpt-4o-mini',

  // Режим локального ИИ (OpenAI-совместимый API)
  LOCAL_AI_URL: 'http://localhost:11434/v1',   // Твой порт
  LOCAL_AI_MODEL: 'ВАШ_ЛОКАЛЬНЫЙ_MODELS_NAME' // Например: llama3, mistral
};
```

**.gitignore обязательно содержит:**
```
config.js
*.env
.env
```

## 4. База данных Supabase

### Таблица: `profile_current` — «Я сейчас» (одна запись)

```sql
CREATE TABLE profile_current (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at      TIMESTAMP DEFAULT NOW(),
  updated_at      TIMESTAMP DEFAULT NOW(),
  q1_description  TEXT,  -- Как я бы описал себя в 3–5 словах?
  q2_habits       TEXT,  -- 3 главные привычки/паттерна
  q3_fears        TEXT,  -- 3 главных страха/ограничения
  q4_strengths    TEXT,  -- Сильные качества, которыми не пользуюсь
  q5_character    TEXT   -- Типичное «я» как персонаж
);
```

### Таблица: `profile_future` — «Кем хочу стать» (одна запись)

```sql
CREATE TABLE profile_future (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at      TIMESTAMP DEFAULT NOW(),
  updated_at      TIMESTAMP DEFAULT NOW(),
  q1_description  TEXT,  -- Идеальный я в 3–5 словах
  q2_habits       TEXT,  -- 3 привычки будущего я
  q3_fears_gone   TEXT,  -- 3 страха, которые должны исчезнуть
  q4_qualities    TEXT,  -- Качества, которые хочу развить
  q5_day_scene    TEXT   -- Сцена из обычного дня будущего я
);
```

### Таблица: `daily_checkins` — ежедневные записи

```sql
CREATE TABLE daily_checkins (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  date            DATE NOT NULL UNIQUE,  -- один чекин в день
  created_at      TIMESTAMP DEFAULT NOW(),
  q1_mood_score   INTEGER CHECK (q1_mood_score BETWEEN 1 AND 10),
  q1_emotions     TEXT,  -- 1–2 слова-эмоции
  q2_pattern      TEXT,  -- Главная мысль/страх/убеждение дня
  q3_actions      TEXT,  -- Что сделал для приближения к желаемому я
  q4_thoughts     TEXT,  -- О чём больше всего думал
  q5_choice       TEXT   -- Осознанный выбор или автопилот
);
```

### Таблица: `monthly_reports` — сводки

```sql
CREATE TABLE monthly_reports (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  year        INTEGER NOT NULL,
  month       INTEGER NOT NULL,  -- 1–12
  created_at  TIMESTAMP DEFAULT NOW(),
  report_text TEXT,              -- Полный текст ИИ-сводки (Markdown)
  UNIQUE(year, month)
);
```

## 5. Интерфейс

### Навигация
Три таба вверху страницы:
- **Чекин** — активен по умолчанию
- **Мой профиль** — блоки «Я сейчас» и «Кем хочу стать»
- **Сводка** — просмотр и генерация месячных отчётов

### Таб «Чекин»
- Показывает сегодняшнюю дату
- Если чекин за сегодня уже есть → отображает заполненные ответы с кнопкой «Редактировать»
- Если нет → форма с 5 вопросами (блок 2)
- Вопросы:
  1. **Настроение дня** — слайдер 1–10 + текстовое поле «1–2 эмоции»
  2. **Главный паттерн дня** — textarea
  3. **Действия к цели** — textarea
  4. **О чём думал** — textarea
  5. **Осознанный выбор** — textarea
- Кнопка «Сохранить чекин» → запись в Supabase
- Маленький календарь текущего месяца: зелёная точка = чекин есть, серая = нет

### Таб «Мой профиль»
- Два раскрывающихся блока: **«Я сейчас»** и **«Кем хочу стать»**
- При первом открытии оба пустые → кнопка «Заполнить»
- После заполнения → отображение ответов + кнопка «Редактировать»
- 5 вопросов в каждом блоке (см. раздел 7)

### Таб «Сводка»
- Список месяцев с уже сгенерированными сводками (кликабельные)
- Блок «Текущий месяц»:
  - Счётчик: «Чекинов за месяц: X из Y дней»
  - Если сегодня 1-е число: кнопка **«Сгенерировать сводку за прошлый месяц»** (активна)
  - В любой другой день: кнопка неактивна, подпись «Сводка генерируется 1-го числа»
  - **Кнопка «Сгенерировать сейчас»** — доступна всегда (для тестирования)
- При генерации: индикатор загрузки + текст «ИИ анализирует твой месяц...»
- Готовая сводка отображается в Markdown-форматировании

## 6. Модуль ИИ (ai.js)

### Переключение режима
```javascript
// Определяется из CONFIG.AI_MODE
async function callAI(prompt) {
  if (CONFIG.AI_MODE === 'local') {
    return await callLocalAI(prompt);
  } else {
    return await callOpenAI(prompt);
  }
}
```

### Промпт для месячной сводки
ИИ получает:
1. Блок «Я сейчас» (заполнен при старте)
2. Блок «Кем хочу стать»
3. Все чекины за месяц (в хронологическом порядке)

**Системный промпт:**
```
Ты — личный коуч и аналитик. Твоя задача — проанализировать месяц жизни человека
и дать честную, конструктивную и тёплую обратную связь на русском языке.
Структурируй сводку по следующим разделам:

1. 📊 Общая картина месяца (2–3 абзаца)
2. ✅ Что улучшилось и выросло
3. ⚠️ Что осталось прежним или стало хуже
4. 🔁 Повторяющиеся паттерны (мысли, эмоции, поведение)
5. 📈 Динамика настроения (на основе числовых оценок)
6. 🎯 Насколько ты приближаешься к желаемому «я»
7. 💡 3 конкретных рекомендации на следующий месяц

Пиши честно, без лишних комплиментов. Используй конкретные примеры из записей.
```

## 7. Вопросы (полный список)

### Блок «Я сейчас» (заполняется один раз)
1. Как я бы описал(а) себя сейчас в 3–5 словах?
2. Какие 3 главные привычки или паттерна сейчас определяют мою жизнь?
3. Какие 3 главных страха или ограничивающих убеждений сейчас задерживают меня?
4. Какие качества у меня сильны, но я ими почти не пользуюсь?
5. Какой человек «здесь и сейчас» я бы назвал(а) своим типичным «я» (как персонаж)?

### Блок «Кем хочу стать» (заполняется один раз)
1. Как бы я описал(а) своего «идеального себя» через 1–3 года в 3–5 словах?
2. Какие 3 привычки и паттерна должен(на) иметь этот человек, которым я хочу стать?
3. Какие 3 страха или убеждения у меня сейчас должны исчезнуть в этом будущем «я»?
4. Какие качества у будущего «я» я ценю больше всего и хочу развить сейчас?
5. Как бы я представил(а) этого будущего себя в одной короткой сцене из обычного дня?

### Ежедневный чекин
1. **Настроение:** Как я себя чувствую сегодня по шкале от 1 до 10? + Какие 1–2 эмоции преобладали?
2. **Паттерн дня:** Какая главная мысль/страх/убеждение крутилось в голове сегодня?
3. **Действия к цели:** Что сделал(а) сегодня, чтобы приблизиться к желаемому я?
4. **Главные мысли:** Какая тема или ситуация занимала мысли чаще всего?
5. **Осознанный выбор:** Какое решение принял(а) осознанно, а не по инерции?

## 8. UX и визуальный стиль

- **Стиль:** Минимализм. Белый/светло-серый фон, тёмный текст, никакой мишуры
- **Шрифт:** System font stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)
- **Акцентный цвет:** один, на выбор разработчика (например, `#2D6A4F` — тёмно-зелёный)
- **Кнопки:** Простые, с hover-эффектом
- **Textarea:** Авторесайз по содержимому
- **Мобильная адаптация:** Обязательна (сайт используется каждый день, часто с телефона)
- **Анимации:** Минимум. Только плавный fade при переключении табов

## 10. Дополнительные требования

- **Оффлайн-защита:** Если Supabase недоступен — показать сообщение об ошибке, не ломать интерфейс
- **Дублирование чекина:** Нельзя создать два чекина за один день. При попытке — предложить редактировать существующий
- **Автосохранение:** Необязательно, но приветствуется (сохранение черновика в localStorage)
- **Загрузка:** Все операции с БД сопровождаются визуальным индикатором (спиннер или текст)
- **Никакого фреймворка:** Только Vanilla JS. Supabase подключается через CDN (`<script>` тег)
- **Supabase CDN:**
```html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
```

## 11. Порядок реализации (рекомендуемый)

1. Создать таблицы в Supabase (SQL из раздела 4)
2. Настроить `config.js` (скопировать из `config.example.js`)
3. Сверстать `index.html` с тремя табами и базовыми стилями
4. Реализовать `supabase.js` — все CRUD-функции для 4 таблиц
5. Реализовать таб «Мой профиль» (чтение/запись двух профилей)
6. Реализовать таб «Чекин» (форма + календарь с точками)
7. Реализовать `ai.js` (оба режима: OpenAI + локальный)
8. Реализовать таб «Сводка» (генерация + список истории)
9. Добавить `.gitignore`, `config.example.js`, `README.md`
10. Проверить на мобильном устройстве
